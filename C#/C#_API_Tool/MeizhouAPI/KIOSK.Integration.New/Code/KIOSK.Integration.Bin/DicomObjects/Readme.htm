This Package

This package is designed for DicomObjects users who have already installed the trial version, and who have now obtained a registered copy. All the files in this zip file should be unzipped into your main DicomObjects directory, over-writing any existing files of the same name. No re-registration should be necessary, but if you have moved or renamed your DicomObjects.ocx previously, then you may need to re-register the ocx by changing the directory containing new new DicomObjects.ocx and then using the commands:

regsvr32 /u DicomObjects.ocx
regsvr32 DicomObjects.ocx

Delphi and C++ Builder Users:

Please ensure that you read and act upon the instructions in the LicenseKey file.

In general, you should re-import your support packages once you have installed and re-registered the OCX and LIC files as above, as they will then incorporate the license information.  If you have problems with this, please contact Medical Connections for assistance.

Important Notes:

  • There are serious bugs in the import routines in the original release of version 6 of Delphi & C++ Builder, which will cause compilation to fail if you import yourself using these versions.  These bugs are fixed in the latest updates, which are highly recommended.
  • Delphi appears to have problems upgrading ActiveX controls properly. If after a DicomObjects upgrade you are experiencing unusual behaviour form a DicomViewer control, please try deleting the control and replacing it with a "new" one. If you do this, you must then remember to re-link the events on the control’s Delphi property sheet.
